# 如何用 iOS 捷徑把策略結果交給 Claude 分析

這份文件是給使用者看的操作說明，不是程式碼。App 會把目前所有股票的策略
比較結果組成文字交給捷徑，捷徑再透過官方「Ask Claude」動作取得分析並顯示。
捷徑只需要在 iOS 的「捷徑」App 設定一次。

## 前置準備

- iPhone 上已安裝「捷徑」App（iOS 內建）
- iPhone 上已安裝並登入 Claude App

## 建議做法：App「Claude 分析」按鈕 + 極簡捷徑（三個動作）

App 首頁的「**Claude 分析**」按鈕會直接執行指定名稱的捷徑，並且**已自動
在文字開頭附上提示詞**（要求 Claude 針對每檔給今日建議動作與理由、用純文字
條列回覆），所以捷徑裡不需要組提示詞、也不需要任何格式轉換。

### 建立捷徑

1. 開啟「捷徑」App，點右上角「+」建立新捷徑
2. 捷徑名稱取「**AI資產分析**」——必須與 App 設定頁「Claude 分析
   捷徑名稱」**完全一致**（預設就是這個名字，照取就不用改設定）
3. 加入三個動作：
   1. 「**接收分享的輸入**」，型別限定「**文字**」，若沒有輸入：繼續
   2. 「**Ask Claude**」（動作庫搜尋 Claude），輸入帶「捷徑輸入」變數
   3. 「**顯示結果**」，內容帶「Ask Claude」的輸出
4. 點「Ask Claude」動作右側的展開箭頭（⌄），把「**執行時顯示**」
   （Show When Run）**關掉**——否則執行時會先彈出一個不能捲動的
   回應預覽框，要按完成才會進到「顯示結果」，等於彈兩次視窗
5. 點右上角「完成」儲存

### 使用

首頁按「Claude 分析」→ 跳到捷徑 App 自動執行 → 彈出分析結果。
因為 App 已要求 Claude 用純文字條列回覆，「顯示結果」直接就是好讀的。

### 注意事項

- 「Ask Claude」使用你 Claude App 裡的**預設模型**
- 透過此動作送出的訊息**計入你的 Claude 用量額度**，額度用完時捷徑會
  失敗，屬正常現象
- 參考官方說明：[Use Claude app intents, shortcuts, and widgets on iOS](https://support.claude.com/en/articles/10263469-use-claude-app-intents-shortcuts-and-widgets-on-ios)

## 備用入口：從分享面板走

App 的「分享」按鈕會開系統分享面板，匯出的是**純策略數據（不含提示詞）**，
適合分享到其他 App 或存檔。若要從這個入口交給 Claude 分析，同一個捷徑
要多設定兩件事：

1. 捷徑設定（ⓘ 或齒輪圖示）裡開啟「**在分享表單中顯示**」
2. 在「接收分享的輸入」與「Ask Claude」之間加一個「**文字**」動作自行組
   提示詞（因為這個入口的文字沒有內建提示詞），範例：

   ```
   你是台股投資顧問。以下是我的持股策略狀態，請針對每檔給出
   「今日建議動作」與理由，最後給整體資產配置提醒，回覆用純文字條列：

   [捷徑輸入]
   ```

   其中 `[捷徑輸入]` 是變數——點輸入框上方的變數列選「捷徑輸入」。
   「Ask Claude」的輸入改帶這個「文字」動作的輸出。

## 選配：想要有排版的顯示（粗體、標題）

「顯示結果」只能顯示純文字。若想要粗體、標題等排版效果，改成：

1. 提示詞改成要求 Claude「請用 Markdown 排版：每檔股票一個標題、建議動作
   用粗體、理由用條列」（適用分享面板入口，在捷徑的「文字」動作裡改；
   一鍵入口的提示詞是 App 內建的純文字版，不適用此變化）
2. 「Ask Claude」之後接「**從 Markdown 製作 RTF**」（輸入帶 Ask Claude
   的輸出），再接「**快速查看**」（輸入帶 RTF）取代「顯示結果」

「從 Markdown 製作 RTF」只是把文字在記憶體裡轉成有排版的格式直接顯示，
**不會**在「檔案」App 留下任何 RTF 檔。

## 備案：沒有 Ask Claude 動作時

動作庫搜不到「Ask Claude」時的通用做法（半自動，需手動貼上）：

1. 加入「**拷貝到剪貼簿**」動作，把收到的文字複製起來
2. 加入「**開啟 App**」動作，選擇 Claude App
3. 開啟 Claude App 後，貼上剪貼簿內容送出（這一步是手動貼上，但前面的
   複製與開啟 App 都已經自動完成）

## 疑難排解

- **「Claude 分析」按鈕報找不到捷徑或沒反應**：捷徑名稱要與 App 設定頁
  完全一致（含空格與大小寫），預設為「AI資產分析」
- **分享面板看不到我建立的捷徑**：確認「接收分享的輸入」的型別有包含
  「文字」，並確認捷徑設定裡「在分享表單中顯示」是開啟的
- **文字被截斷或亂碼**：本 App 匯出的是純文字（UTF-8），如果在捷徑裡看到
  異常字元，先確認「接收分享的輸入」動作沒有被其他型別轉換動作影響
