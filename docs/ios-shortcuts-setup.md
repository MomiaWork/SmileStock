# 如何在 iOS 捷徑 App 建立「分享策略比較結果給 Claude」的自動化

這份文件是給使用者看的操作說明，不是程式碼。App 這邊只負責把目前所有股票的
策略比較結果組成一段好讀的文字，透過系統分享面板（Share Sheet）分享出去；
剩下「分享面板選捷徑 → 捷徑把文字轉交 Claude App → 取得 AI 分析意見」這段，
是你在 iOS 的「捷徑」App 手動設定一次即可，之後每次分享都會自動照這個流程走。

## 前置準備

- iPhone 上已安裝「捷徑」App（iOS 內建）
- iPhone 上已安裝 Claude App

## 步驟一：建立一個新捷徑

1. 開啟「捷徑」App
2. 點右上角「+」建立新捷徑
3. 幫捷徑取個好記的名字，例如「用 Claude 分析持股」

## 步驟二：讓這個捷徑能出現在分享面板

1. 在捷徑編輯畫面，加入動作「**接收分享的輸入**」（Receive Shared Input，或稱
   「接收 [輸入內容]」）
2. 把接收的型別限定成「**文字**」（Text），這樣分享面板才會在分享文字時把
   這個捷徑顯示出來
3. 點捷徑名稱旁的設定（ⓘ 或齒輪圖示）確認「在分享表單中顯示」已開啟

## 步驟三：組合提示詞

分享進來的只是策略數據，先包一層指令，Claude 才知道要做什麼：

1. 加入「**文字**」動作，內容範例：

   ```
   你是台股投資顧問。以下是我的持股策略狀態，請針對每檔給出
   「今日建議動作」與理由，最後給整體資產配置提醒，回覆精簡條列：

   [捷徑輸入]
   ```

2. 其中 `[捷徑輸入]` 是變數——點輸入框上方的變數列，選「捷徑輸入」
   （即步驟二接收到的分享文字）

## 步驟四：用「Ask Claude」動作取得分析（方式 A，建議）

Claude App 有提供官方捷徑動作「**Ask Claude**」，可以在捷徑內直接送出
訊息並把回覆傳給下一個動作：

1. 在動作庫搜尋「Claude」，加入「**Ask Claude**」動作
   （需要已安裝並登入 Claude App）
2. 把輸入設成上一步「文字」動作的輸出變數
3. 加入「**顯示結果**」動作，內容帶入「Ask Claude」的輸出——
   捷徑跑完會直接彈出 Claude 的分析結果

注意事項：

- 「Ask Claude」使用你 Claude App 裡的**預設模型**
- 透過此動作送出的訊息**計入你的 Claude 用量額度**，額度用完時捷徑會
  失敗，屬正常現象
- 參考官方說明：[Use Claude app intents, shortcuts, and widgets on iOS](https://support.claude.com/en/articles/10263469-use-claude-app-intents-shortcuts-and-widgets-on-ios)

**方式 B（動作庫搜不到 Ask Claude 時的通用備案）**

1. 加入「**拷貝到剪貼簿**」動作，把步驟三組好的文字複製起來
2. 加入「**開啟 App**」動作，選擇 Claude App
3. 開啟 Claude App 後，貼上剪貼簿內容送出即可（這一步是手動貼上，但前面的
   複製與開啟 App 都已經自動完成）

## 步驟五：儲存捷徑

完成後點右上角「完成」儲存。

## 之後怎麼用

1. 打開這個 App，確認股票清單、策略都設定好
2. 按下「分享比較結果」按鈕，開啟系統分享面板
3. 在分享面板選擇你剛剛建立的捷徑（例如「用 Claude 分析持股」）
4. 捷徑自動執行：組合提示詞 → Ask Claude 送出 → 直接彈出分析結果，
   全程不需要手動貼上（方式 B 則需在 Claude App 內手動貼上送出）

## 疑難排解

- **分享面板看不到我建立的捷徑**：回到步驟二，確認「接收分享的輸入」的型別
  有包含「文字」，並確認捷徑設定裡「在分享表單中顯示」是開啟的
- **文字被截斷或亂碼**：本 App 匯出的是純文字（UTF-8），如果在捷徑裡看到
  異常字元，先確認「接收分享的輸入」動作沒有被其他型別轉換動作影響
